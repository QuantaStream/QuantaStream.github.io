Open-source HTAP engine

Fresh analytics on compressed bitmap storage.

QuantaStream is the rebranded Quanta project: a distributed database engine designed for low-latency analytical queries over continuously ingested data. It uses Roaring bitmaps for compact storage and serves clients through a MySQL-compatible proxy.

What it is built to do

HTAP patterns without pretending to be a full relational database.

QuantaStream focuses on fast analytics over fresh, compressed data. It does not currently provide transactions, stored procedures, or the full MySQL feature set. The design trades those features for bitmap-oriented query execution and streaming-aware ingestion.

HTAP

Operationally fresh analytics

Built for workloads that need analytical reads over data that is still arriving, with ingestion and query paths designed as part of the same system.

Roaring bitmaps

Compressed storage and execution

Data is compressed on import and queried in that representation. Query Processors rehydrate serialized bitmaps and apply operations such as AND, OR, and difference.

MySQL proxy

Familiar client access

The proxy emulates a subset of the MySQL networking protocol, opening the door to standard drivers while keeping SQL support intentionally scoped.

Data Nodes

Distributed storage layer

Data Nodes serve bitmap data over gRPC and store high-cardinality strings in a distributed, persistent hashtable across the cluster.

Streaming

Buffered ingestion pipeline

Kinesis consumers transform incoming records into bitmaps, aggregate updates with OR operations, and write the results to the appropriate Data Nodes.

Discovery

Consul-backed coordination

Consul is used for Data Node discovery and schema metadata, giving query and ingestion services a consistent view of active endpoints and table definitions.

Architecture

A small set of services with clear jobs.

The original Quanta README describes a distributed design influenced by Cassandra-style scalability and fault tolerance. QuantaStream presents that architecture under the new name while keeping the underlying claims tied to the current implementation and roadmap.

Client applications

Applications connect using MySQL-compatible drivers and tools through a network load balancer.

Query Processor

Multiple proxy instances can serve SQL connections, fetch compact bitmap responses from Data Nodes, and combine results for the client.

Data Nodes

The storage and retrieval tier handles bitmap data and the distributed hashtable used for high-cardinality strings.

Kinesis Consumers

Consumers translate streaming events into bitmap updates and pre-aggregate them before writing to storage.

Consul

Service discovery and schema metadata live in Consul, making active nodes and field definitions available to upstream services.

Roadmap

Practical next steps, not finished-product promises.

The project README identifies version 0.8 as alpha. The items below summarize planned work from the source material and should be read as direction rather than current capability.

SQL

Broader analytical queries

GROUP BY, HAVING, multiple aggregations, SQL subqueries, temporary tables, views, and more join patterns are listed as future work.

Operations

Cluster administration

Monitoring tools, APIs, and the ability to add or remove Data Nodes in a running cluster are called out for beta-era work.

Scaling

Autoscaling and balancing

Planned work includes metrics-driven autoscaling and improved data distribution based on resource utilization.

Resilience

Active/active HA and DR

The roadmap includes multi-data-center high availability and disaster recovery, with conflict-resolution strategies still to be designed.

Governance

RBAC interfaces

Authentication is currently based on OpenID Connect JWT tokens, with RBAC interfaces identified as a future area.

Performance

GPU exploration

GPU-accelerated bitmap processing is described as an exploration path for large-scale data processing.

What QuantaStream is today

An open-source alpha database engine for bitmap-oriented HTAP experimentation, with MySQL-compatible access and streaming ingestion concepts already central to the design.

What it is not claiming

It is not presented here as a drop-in MySQL replacement, a transactional database, or a mature production platform with every roadmap item complete.
